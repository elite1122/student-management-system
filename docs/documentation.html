1. Project Overview

The Student Management System is a full-stack web application that allows administrators to manage student records efficiently. It provides a clean, modern interface to perform all CRUD (Create, Read, Update, Delete) operations on student data, backed by a RESTful API connected to a cloud-hosted MongoDB database.

The system is fully containerized using Docker and features an automated CI/CD pipeline powered by GitHub Actions that deploys both the frontend and backend to Vercel on every push to the main branch.

Purpose: Provide a centralized system to add, update, delete, and view student records with a professional, responsive UI and scalable backend architecture.

2. Features

Core Functionality

  • Add new students with full profile information
  • Edit existing student records
  • Delete students with a confirmation dialog
  • View all students in a sortable, paginated table
  • Search students by name, email, or course
  • Filter students by status (Active, Inactive, Graduated, Suspended)

Dashboard

  • Real-time stats cards: Total, Active, Inactive, Graduated students
  • Pagination with page navigation
  • Live status indicator

UI / UX

  • Professional, light-themed design with modern typography
  • Smooth animations powered by Framer Motion
  • Fully responsive layout (mobile & desktop)
  • Toast notifications for all actions (success/error)
  • Hover-reveal action buttons in the table

9. Local Development Setup

Prerequisites

  • Node.js v20 or higher
  • npm v9 or higher
  • MongoDB Atlas account (free tier available)
  • Git

Step 1 — Clone the Repository

git clone https://github.com/elite1122/student-management-system.git
cd student-management-system

Step 2 — Setup Backend

cd backend
npm install
# Create .env file with your MongoDB URI
npm run dev
# Server runs on http://localhost:5000

Step 3 — Setup Frontend

cd frontend
npm install
# .env already set to VITE_API_URL=http://localhost:5000/api
npm run dev
# App runs on http://localhost:3000
Open http://localhost:3000 in your browser to use the application.

10. Docker & Containerization

The application is fully containerized. Each service has its own Dockerfile and they are orchestrated together using docker-compose.yml.

Backend Dockerfile

Uses a multi-stage build — installs dependencies in a builder stage, then copies only the production files to a lean final image. Runs as a non-root user for security.

FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

FROM node:20-alpine
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY . .
USER appuser
EXPOSE 5000
CMD ["node", "server.js"]

Frontend Dockerfile

Builds the React app with Vite, then serves the static output using Nginx.

FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

Docker Compose

Connects both containers on a shared internal network. The frontend proxies /api requests to the backend via Nginx.

Running with Docker

1
Install Docker Desktop and ensure it is running.
2
Create root .env file with MONGO_URI=your_connection_string
3

Job 1 — Deploy Backend

  1. Checkout repository code
  2. Setup Node.js 20
  3. Install Vercel CLI globally
  4. Deploy backend to Vercel (production) using project secrets

Job 2 — Deploy Frontend

  1. Checkout repository code
  2. Setup Node.js 20
  3. Install npm dependencies
  4. Build the React app with VITE_API_URL injected
  5. Install Vercel CLI globally
  6. Deploy frontend to Vercel (production)
Note: Job 2 only starts after Job 1 completes successfully (needs: deploy-backend). This ensures the backend is always deployed before the frontend.

12. Deployment (Vercel)

Both services are deployed to Vercel as separate projects.

Backend Deployment

The Express backend is deployed as a Vercel Serverless Function. The vercel.json routes all incoming requests to server.js. MongoDB connections are cached between function invocations for performance. 

{
  "version": 2,
  "builds": [{ "src": "server.js", "use": "@vercel/node" }],
  "routes": [{ "src": "/(.*)", "dest": "server.js" }]
}

Frontend Deployment

The React app is built by Vite into a static bundle and hosted on Vercel's CDN. The vercel.json rewrite rule ensures all routes serve index.html for React Router compatibility. 

{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}

14. Conclusion
